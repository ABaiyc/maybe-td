# 计划：塔防「不知道能不能完成的塔防游戏」首个内容版本

> 状态：草案，可能继续修改。等用户明确指示再开始实现。
> 主题：**猪猪打狼**（猪拿武器当塔，狼当敌人）。塔设计源文件见 `docs/塔设计-30种.md`（权威设计稿）。

## Context（背景与目标）

已验证可玩切片（固定塔位 + 单体塔 + 5 波 + 胜负，Godot 4.6 无头跑通全 5 波）跑通了核心循环。
现在进入"首个有主题、有深度、有爽感的内容版本"。

**本期范围（用户限定）**：2 个关卡、确定性 30 塔融合系统（先做 3 个基础塔可玩起来）、防御塔图鉴、美术先用免费像素资源（暂定像素风、可换）。
**世界观**：**猪（塔）打狼（敌人）**。猪手持不同武器布防，狼群沿路径进攻。两个 Boss——披着羊皮的狼（伪装躲避自动索敌）、头狼（狡猾，突然改走备用路径）。
**核心体验诉求**：拖拽合成升级（确定性、有深度）+ 强打击感/视觉爽感。

设计纪律沿用 game-generation skill 的 **build→replay→evaluate**：每个里程碑用无头自动战斗验证可启动+无运行时报错，再做下一步。美术保持**数据驱动、可热替换**（贴图路径在 def 里，缺图回退到代码绘制）。

## 设计

### A. 防御塔系统（30 种确定性融合，权威细节见 `docs/塔设计-30种.md`）

**主题映射**：每个塔 = 一只猪举着对应武器。三个基础元素：
| 元素 | 基础塔（猪） | 角色 |
|---|---|---|
| **激光 L** | 激光猪 | 精准·穿透·持续单体伤害；攻速慢、不追踪 |
| **元素 E** | 元素猪 | 抛射元素球范围爆炸，随机燃烧/冰冻/麻痹；单发低、弹速慢 |
| **实弹 B** | 机枪猪 | 高速连射稳定物理伤害；无穿透、打不动高甲 |

**等级与融合规则（核心）**：
- 同级才能融合，融合后等级 +1，**最高三级**（融合两次）。
- 拖拽合并：按住一塔拖到另一**同级**塔上松手 → 合成；源塔位清空，结果落目标塔位。
- **确定性、无随机变异**（旧计划的 12% 变异已移除——与"同成分不同路径=不同塔"的确定系统冲突）。
- **关键机制：同样成分、不同融合路径 → 不同三级塔**（纯+纯 ≠ 混合+混合）。所以融合表按**二级塔的无序配对**做查表，而非只看元素计数。
- 三级塔 = **机制质变**（领域/分裂/连锁/塔间组网/集火标记等），不是数值堆叠。

**完整融合表**（实现的权威依据，编码进 `data/tower_defs.gd`）：

一级→二级（3 基础两两配对 = 6 种）：
`L+L→聚焦激光(LL)` · `E+E→元素风暴(EE)` · `B+B→机枪塔(BB)` · `L+E→元素激光(LE)` · `L+B→穿甲弹(LB)` · `E+B→榴弹炮(EB)`

二级→三级（6 种二级两两配对 = **21 种，每种唯一**）：
| 配对 | 三级塔 | 配对 | 三级塔 |
|---|---|---|---|
| LL+LL | 棱镜塔 | EE+EE | 元素反应炉 |
| BB+BB | 轨道炮 | LL+LE | 光束爆裂炮 |
| LL+LB | 高速穿甲枪 | EE+LE | 元素矩阵 |
| EE+EB | 弹幕炮塔 | BB+LB | 反器材狙击炮 |
| BB+EB | 爆裂榴弹炮 | LL+EE | 等离子场 |
| LE+LE | 双子激光塔 | LL+BB | 精准狙击塔 |
| LB+LB | 连射穿甲炮 | EE+BB | 自动布雷车 |
| EB+EB | 元素榴弹阵列 | LE+LB | 电磁脉冲塔 |
| LL+EB | 激光引爆炮 | LE+EB | 催化反应塔 |
| EE+LB | 元素穿甲弹 | LB+EB | 全能作战塔 |
| BB+LE | 战术标记炮 | | |

> 注：6 种二级塔的带重复无序配对恰好 21 组，与 21 种三级塔一一对应——融合表完备无歧义。每塔的攻击/特性/定位以 `docs/塔设计-30种.md` 为准。

### B. 敌人（狼群）+ 两个 Boss（沿用既定机制，重新主题化为狼）
- **基础狼 3 种**：野狼（均衡）、疾风狼（高速低血）、铁背狼（高血+减伤，逼玩家上穿甲 LB / 范围）。数据在 `data/enemy_defs.gd`。
- **Boss① 头狼（改路）**：血量阈值（每掉 25%）或定时，跳到该关**备用分支路径**并重算最近路点，考验布防覆盖面。
- **Boss② 披着羊皮的狼（伪装）**：默认**伪装态**——自动索敌的塔**无法锁定**它；但 **E系（元素）的范围/领域/地雷类伤害无需锁定，仍能命中**，是它的天敌。没有 E 系它会大摇大摆穿过，教学玩家组建元素体系。

### C. 两个关卡（数据驱动 + 进度）
- `data/levels.gd`：每关含 `waypoints`（+1~2 条备用分支供 Boss① 用）、`slot_positions`、波次脚本、Boss 配置、主题配色/贴图集。
- **关卡1 草原村道**：基础狼为主，结尾 Boss①（头狼）。
- **关卡2 黑森林/雪原**：更硬狼群 + 铁背狼，结尾 Boss②（披着羊皮的狼）。
- 进度：通关1解锁2；含简单关卡选择/胜利结算界面。`GameState` 记录当前关与进度。

### D. 防御塔（猪）图鉴 【新增】
- 一个可从主菜单/关内打开的**图鉴界面**（`scenes/codex.tscn` + `scripts/codex.gd`），数据由 `data/tower_defs.gd` 自动生成。
- 按等级分区展示全部 30 只猪塔：名称、元素成分、**合成配方**（由哪两塔融合而来）、攻击/特性/定位文案、立绘（缺图回退代码绘制）。
- **解锁追踪**：玩家首次获得某塔即点亮，未解锁显示剪影 + "?"。已点亮的显示完整信息。
- 解锁记录持久化到 `user://` 存档（JSON），跨局保留。`GameState` 维护 `discovered` 集合与读写。

### E. 爽感系统（四项全做）
| 系统 | 实现要点 |
|---|---|
| **顶帧 + 震屏** | 场景加 `Camera2D`；trauma 衰减式 shake（`camera_shake.gd`）；命中/击杀短暂 hitstop（`Engine.time_scale` 微降+定时恢复，谨慎用于大命中/击杀） |
| **粒子血爆** | 击杀/命中触发一次性 `GPUParticles2D`（像素方块碎裂、发光拖尾），按狼配色 |
| **伤害数字** | 池化 `damage_number.gd`（飘起淡出，暴击更大更黄） |
| **连击 + Boss 慢镜** | `GameState` 记连杀窗口与倍率，HUD 显示 combo + 音效升调；击杀 Boss 触发 `time_scale` 慢动作 + 镜头拉近 + 闪白 |
| 基础音效 | 接 Kenney/freesound CC0：开火/命中/击杀/合成/Boss，作占位可后换 |

### F. 美术（像素占位、可热替换）
- 每个 tower/enemy/boss def 带 `sprite` 路径字段；**缺图回退到 `_draw()` 代码绘制**，保证任何阶段都能跑。
- **省美术策略**：猪塔 = 「猪身 + 武器叠加层」，等级靠换色/放大/特效区分；30 塔不必 30 张全新立绘。图鉴展示合成后的猪+武器组合。
- 免费来源：itch.io / Kenney 的 CC0 像素猪/狼/武器包；找不到就用程序生成的像素占位精灵。
- 配色：猪=暖粉/黄，狼=冷灰/黑，披羊皮狼=伪装白（揭示后变灰黑）。

### G. 游戏流程与胜负机制 【新增】
**单局流程**：
1. 点击「开始」→ 进入**准备阶段**：10s 倒计时。
2. 准备阶段：点击可放塔格子 → 选择要放置的基础塔布防（可放置/合成，给玩家从容布局）。
3. 倒计时结束 → **战斗阶段**：狼群按波次出现。
4. 战斗中**全程**可对塔进行 **拆除 / 合成 / 放置**（拆除返还部分金币）。
5. 全部波次清空、场上无狼且国王存活 → **战斗结束（结算）**。

**终点 = 大肥猪国王（胜负核心，取代抽象"生命值"）**：
- 路径终点是**大肥猪国王**（带血条 `king_hp`）。
- 狼**走到终点不消失**，而是进入「围攻」状态聚集在国王身边，**持续攻击国王**掉血；靠近终点的塔仍可击杀围攻的狼（鼓励玩家在终点附近布防补救）。
- **失败**：`king_hp` 见底。
- **胜利**：扛过全部波次后，场上围攻的狼被清光且国王存活。
- 落地：`GameState` 用 `king_hp` 取代 `lives`，并加**阶段状态机（准备/战斗/结算）+ 10s 倒计时**；新增国王实体（`scripts/king.gd`）与「围攻狼」行为（到达终点→停下→定时攻击国王，仍可被塔击杀）。

## 验证方法

- **启动门槛（每个 M）**：`godot --headless --import` 建类缓存 → `godot --headless --fixed-fps 60 --quit-after N` 跑临时 `_debug_autoplay`，确认零解析错误、零运行时报错、关卡能跑到胜负。
- **融合表回放**：脚本驱动建塔/合成，**断言全部 6 个二级配对 + 21 个三级配对产出正确塔**（融合系统是本期最易出错处，必须逐项断言）。
- **机制回放**：断言伪装狼仅被 E 系命中、头狼改路后仍被索敌、**狼到终点进入围攻且 `king_hp` 正确扣减、国王死判负/清场判胜**、图鉴解锁状态正确。
- **爽感/美术（交互）**：实机运行截图（命中粒子、伤害数字、震屏、Boss 慢镜、像素精灵），对照"机制≥视觉反馈>内容>美术"自评。
- 每个里程碑完成后写入 `修改日志.md` 未发布区并提交（走 PR 流程，`main` 已配分支保护）。

## 假设与待定（实现中按需微调，不阻塞开工）

- **变异机制移除**——若用户仍想要"合成惊喜"，需另行约定如何与确定性融合表共存。
- Boss② 伪装的"克星"定为 **E 系范围伤害**（因其无需锁定）；若想用专门"侦测塔"，需在 30 塔体系外另加，待定。
- 连击窗口约 2s、塔位数量、各塔具体数值首版给手感占位，后续凭实测调；逐塔数值以 `docs/塔设计-30种.md` 的定性描述为准，定量由实现时给出。
- 关卡用**数据配置**而非各自独立场景（共用 `main`/`level` 渲染），利于加第三关。
- 音效为 CC0 占位，非本期重点；找不到合适像素猪/狼素材则用程序生成占位精灵。
