# 计划：免疫塔防「不知道能不能完成的塔防游戏」首个内容版本

> 状态：草案，可能继续修改。等用户明确指示再开始实现。

## Context（背景与目标）

本次已验证可玩切片（固定塔位 + 单体塔 + 5 波 + 胜负，Godot 4.6 无头跑通全 5 波）跑通了核心循环。
现在从"原型"进入"首个有主题、有深度、有爽感的内容版本"。

**本期范围（用户明确限定）**：2 个关卡、3 种基础防御塔、美术先用免费像素资源（暂定像素风、后续可换）。
**世界观**：免疫细胞（塔）防御病毒（敌人），路径是血管/神经，两个 Boss——癌细胞（伪装躲避某类塔）、病毒王（变异突改路径）。
**核心体验诉求**：拖拽合成升级 + 变异惊喜 + 强打击感/视觉爽感。

设计纪律沿用 game-generation skill 的 **build→replay→evaluate**：每个里程碑用无头自动战斗验证可启动+无运行时报错，再做下一步。美术保持**数据驱动、可热替换**（贴图路径在 def 里，缺图回退到代码绘制），以兼容"像素风后续可能更改"。

## 设计

### A. 三种基础免疫细胞塔（T1）+ 角色分工
| 代号 | 角色 | 关键参数 | 特性 |
|---|---|---|---|
| **中性粒细胞 N** | 单体高射速 DPS | 远射程、快、低伤 | 主输出，便宜 |
| **巨噬细胞 M** | 近距溅射/吞噬 AoE | 短射程、慢、高伤、溅射 | 清群 |
| **树突状细胞 D** | 减速光环 + **侦测** | 中射程、减速、低伤 | **揭示隐身**（克制癌细胞伪装） |

### B. 合成系统（拖拽合并 + 品级 + 特殊 + 变异）
- **交互**：按住一个塔拖到另一塔位上松手触发合成；源塔位清空，结果落在目标塔位。
- **同种升级**：N+N→N2，N2+N2→N3（M、D 同理）。每级数值跃升 + 贴图换色/放大。
- **跨种特殊（少量，控制美术量）**：N+M→**T杀伤细胞**（高单体+小溅射）；M+D→**呈递巨噬**（大 AoE+揭示光环）；N+D→**NK细胞**（高速+对隐身暴击+揭示）。
- **变异**：每次合成约 **12%** 概率，产出非预期结果——随机一种"异种"塔或稀有"干细胞"通配单位（下次合成可变任意种）。是惊喜点也是爽点。
- 合成规则与数值集中在 `data/tower_defs.gd`（目录字典 + 合成表 + 变异表），便于调平衡。

### C. 病毒（敌人）种类 + 两个 Boss
- **基础病毒 3 种**：普通（均衡）、快速（高速低血）、包膜（高血/减伤，逼玩家上 AoE/特殊）。数据在 `data/enemy_defs.gd`。
- **Boss① 病毒王（变异改路）**：血量阈值（如每掉 25%）或定时触发，跳到该关**备用分支路径**，重算最近路点。考验布防覆盖面。
- **Boss② 癌细胞（伪装）**：默认**隐身**——只有进入 D 系/NK 的揭示光环内才可被锁定攻击（NK 始终可见）。没有侦测塔它会大摇大摆穿过，教学玩家必须合成/布置 D。

### D. 两个关卡（数据驱动 + 进度）
- `data/levels.gd`：每关含 `waypoints`（+1~2 条备用分支供 Boss① 用）、`slot_positions`、波次脚本、Boss 配置、主题配色/贴图集。
- **关卡1 血管**：基础病毒为主，结尾 Boss①（病毒王）。
- **关卡2 神经**：更硬病毒 + 包膜，结尾 Boss②（癌细胞）。
- 进度：通关1解锁2；含简单关卡选择/胜利结算界面。`GameState` 记录当前关与进度。

### E. 爽感系统（四项全做）
| 系统 | 实现要点 |
|---|---|
| **顶帧 + 震屏** | 场景加 `Camera2D`；trauma 衰减式 shake（`camera_shake.gd`）；命中/击杀短暂 hitstop（`Engine.time_scale` 微降+定时恢复，谨慎用于大命中/击杀） |
| **粒子血爆** | 击杀/命中触发一次性 `GPUParticles2D`（像素方块碎裂、发光拖尾），按敌人配色 |
| **伤害数字** | 池化 `damage_number.gd`（飘起淡出，暴击更大更黄） |
| **连击 + Boss 慢镜** | `GameState` 记连杀窗口与倍率，HUD 显示 combo + 音效升调；击杀 Boss 触发 `time_scale` 慢动作 + 镜头拉近 + 闪白 |
| 基础音效 | 接 Kenney/freesound CC0：开火/命中/击杀/合成/Boss，作占位可后换 |

### F. 美术（像素占位、可热替换）
- 每个 tower/enemy/boss def 带 `sprite` 路径字段；**缺图回退到当前 `_draw()` 代码绘制**，保证任何阶段都能跑。
- 免费来源：itch.io / Kenney 的 CC0 像素"微生物/细胞/病毒"包；找不到合适的就用我生成的简单像素精灵占位。
- 风格统一靠发光描边 + 配色板（免疫=冷蓝/青，病毒=暖红/紫，癌=灰绿伪装）。

## 验证方法

- **启动门槛（每个 M）**：`godot --headless --import` 建类缓存 → `godot --headless --fixed-fps 60 --quit-after N` 跑临时 `_debug_autoplay`，确认零解析错误、零运行时报错、关卡能跑到胜负。
- **机制回放**：自动脚本驱动建塔/合成/放怪，断言品级提升、变异触发、隐身仅在揭示光环内可被锁定、Boss 改路后仍被索敌。
- **爽感/美术（交互）**：实机运行截图（命中粒子、伤害数字、震屏、Boss 慢镜、像素精灵），对照"机制≥视觉反馈>内容>美术"自评。
- 每个里程碑完成后写入 `修改日志.md` 未发布区并提交（走 PR 流程，`main` 已配分支保护）。
